# 앱인토스 다크패턴 방지 정책 검수 결과

토스 앱인토스 [다크패턴 방지 정책](https://developers-apps-in-toss.toss.im/design/consumer-ux-guide.md)에 따른 프로젝트 검수 결과입니다.

---

## 1. 서비스 진입 시 바텀시트 금지 ✅

**정책**: 서비스 진입 직후 전면을 가로막는 광고/알림 동의 바텀시트 노출 금지.

**검수 결과**:
- **진입 흐름**: 스플래시 → 인증(verify) → 프로필 설정(profile-setup) → 메인(home). 진입 시 자동으로 뜨는 바텀시트 없음.
- **메인 레이아웃** (`app/(main)/layout.tsx`): 하단 네비만 노출, 시트/모달 자동 오픈 없음.
- **useEffect로 시트/모달 자동 오픈**: 없음.

**결론**: 준수.

---

## 2. 뒤로 가기 시 바텀시트 금지 ✅

**정책**: 뒤로 가기 시 이전 화면을 막는 알림 동의 등 바텀시트 노출 금지.

**검수 결과**:
- `router.back()` / `router.push("/home")` 등 일반 라우팅만 사용.
- `popstate` / `beforeunload` 등으로 뒤로가기 가로채서 시트 띄우는 처리 없음.

**결론**: 준수.

---

## 3. 나갈 수 있는 선택지 제공 ✅

**정책**: CTA만 강요하지 않고, 거절/닫기 등 나갈 수 있는 선택지 필수.

**검수 결과**:
- **마이페이지** 로그아웃/탈퇴 다이얼로그: "닫기" + "로그아웃"/"탈퇴" 제공.
- **친구 페이지** 삭제/차단 확인: "닫기" + "삭제"/"차단" 제공.
- **글 상세** 삭제/신청 삭제 확인: "닫기" + 실행 버튼 제공.
- **글 상세 신청 폼**: "닫기" + "신청하기" 제공.
- **바텀시트**: 친구 선택(create-post, post-detail), 위치 선택(profile-setup) — 오버레이 클릭으로 닫기 가능. create-post·post-detail 친구 선택 시트에는 **"닫기" 버튼**을 두어 나가기 선택지를 명시함.

**결론**: 준수 (필요 시 닫기 버튼 추가 반영 완료).

---

## 4. 예상하지 못한 순간의 광고 노출 금지 ✅

**정책**: 사용 흐름 중 갑자기 전면 광고 노출 금지.

**검수 결과**:
- 인앱 광고 컴포넌트/전면 광고 노출 로직 없음.

**결론**: 준수.

---

## 5. CTA로 다음 행동 예측 가능해야 함 ✅

**정책**: 버튼만 보고도 다음에 어떤 화면/행동이 이어지는지 알 수 있어야 함. 모호·과장·중복 보조 문구 지양.

**검수 결과**:
- **인증**: "인증 코드 보내기" → 코드 전송, "인증하기" → 인증, "프로필 설정하기" → 프로필 설정 화면으로 이동. 명확함.
- **프로필 설정**: 마지막 단계 CTA를 **"시작하기!"** → **"완료하고 메인으로"** 로 변경하여, 완료 후 메인으로 간다는 점이 드러나도록 수정함.
- **글 작성**: "게시하기" → 게시 동작. 명확함.
- **글 상세**: "미팅 신청하기" → 신청 폼 오픈, "신청하기" → 신청 제출. 명확함.
- 기타 확인/취소 버튼: "닫기", "삭제", "차단" 등 행동이 드러나도록 구성됨.

**결론**: 준수 (프로필 설정 CTA 문구 개선 반영 완료).

## 유지 시 권장 사항

- **알림 동의**: 추후 알림 권한 요청 시, 서비스 진입 직후·뒤로가기 시가 아닌, 알림 설정 화면 등 사용자 의도와 맞는 맥락에서만 요청할 것.
- **바텀시트/모달**: 새로 추가 시 진입·뒤로가기 시 자동 오픈하지 말 것. 항상 "닫기"/"취소" 등 나가기 선택지 제공할 것.
- **CTA**: 새 버튼 추가 시 "다음에 무엇이 일어나는지" 문구로 표현할 것.
